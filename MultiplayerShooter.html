Multiplayer Shooter Game

What I

Learned

Networking In Unreal Engine 5

The main reason I started developing this game was to learn how to implement multiplayer functionality in Unreal Engine 5. While developing this game I learnt a lot about replication, the listen server model and server-client architecture. What I found very interesting and challenging was figuring out when I had to use replication, whether I should use RPCs or replicated variables and how to minimize network traffic. Though I was never planning on releasing this game, I did my best to try to abide to keeping the server authoritative and the client as dumb as possible.

Client Side Prediction

Another interesting concept I learned about was client side prediction. This is a technique used to make the game feel more responsive to the player. The idea is that the client predicts the outcome of the player's actions and then corrects itself if necessary when it receives the server's response. Instances where this technique is applicable include anything that needs to be replicated but has to feel responsive for the player, without breaking the server's authority. Reloading a weapon is a good example of this, as you can update your HUD and correct it if necessary when the server responds.

Server Side Rewind

Perhaps the most challenging, yet most interesting, concept I learned about was server side rewind. This is a lag compensation technique used to make the game feel more responsive to the clients and also more fair. I implemented this in the shooting mechanic of the game. When a player shoots, if SSR is enabled for his weapon then the server will rewind the player's position to the time of the shot and then simulate the shot from its' perspective. This way, the server can determine if the shot was a hit or a miss and then send the result back to the client.

Animation Blending

Since I could only use animations that I got from the Unreal Engine Marketplace, I had to make the most out of them. I improved my understanding of the animation state machine and how to blend animations together. I also learned how to replicate animations, using already replicated player, since the animation instance is not replicated by default.

What I

Would Add

Anti Cheat Detection

Another interesting concept I would like to get into is anti cheat detection. I would start by making sure that all the important game logic is server authoritative, because right now the client has authority over some things that can be exploited by memory address manipulation. For example the fire delay for the weapons is client sided.

Networking Improvements

I would like to add a number of improvements regarding the networking. For example a lobby system where instead of clicking join and being put into a random game, you would see a list of all available lobbies, their players your ping to them, as well as the game mode and map.

A friend system would also be something interesting to research since it would require some sort of account system. Perhaps this could be done using the Steam API, which I have never used before.

More Game Modes

Since the purpose of this project was to learn about networking, I didn't spend much time on the actual game but rather on the networking aspect of it, while also pursuing any interests I had in Unreal Engine that I didn't have the chance to explore yet. If I had the chance to continue working on this project, I would implement various game modes, such as team deathmatch, capture the flag, king of the hill, push the payload etc.
